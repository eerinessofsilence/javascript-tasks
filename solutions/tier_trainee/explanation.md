# Пояснения к решениям (tier_3)

Ниже кратко объяснена логика каждого решения из `solutions/tier_3/solutions.js`.

## 1 уровень

### 1. Приоритет операторов (`expressionResults`)

- Считаются два варианта выражения:
  - `a + b * c` (обычный приоритет операторов).
  - `(a + b) * c` (принудительный приоритет через скобки).
- Результат возвращается объектом.

### 2. Строка из переменных (`profileString`)

- Используется шаблонная строка с подстановкой значений.
- Получается один готовый текст для вывода.

### 3. Типы значений (`detectTypes`)

- Для каждого элемента массива берётся `typeof`.
- Это показывает динамическую типизацию JavaScript.

### 4. Сумма строковых чисел (`sumNumericStrings`)

- Значения преобразуются через `Number(...)`.
- Если хотя бы одно значение нечисловое (`NaN`), возвращается `"Некорректный ввод"`.
- Иначе возвращается числовая сумма.

### 5. Операции со счётчиком (`applyCounterSteps`)

- Последовательно применяются:
  - `+= 5`
  - `*= 2`
  - унарное уменьшение на 1 (эквивалентно `counter--`)
- Возвращается итоговое значение.

### 6. Присваивание с операцией (`addThenDecrement`)

- Число `a` увеличивается на `b` через `+=`.
- После этого уменьшается на `1`.

## 2 уровень

### 1. Добавление и удаление свойств (`manageUserObject`)

- Создаётся объект пользователя.
- Добавляются новые поля (`isStudent`, `email`).
- Поле `age` удаляется через `delete`.

### 2. Вложенные свойства (`setUserAddress`)

- Если `address` отсутствует, сначала создаётся пустой объект.
- Затем записываются `city` и `zip`.

### 3. Сокращённая запись и метод (`createProduct`)

- Используется сокращённый формат свойств: `title, price, inStock`.
- Метод `getInfo` использует `this` и возвращает строку с данными товара.

### 4. Методы объекта (`createAccount`)

- `deposit` пополняет баланс только положительной суммой.
- `withdraw` проверяет корректность и достаточность средств.
- Оба метода читают и изменяют состояние через `this.balance`.

### 5. Глобальный объект `Math` (`mathReport`)

- Через `Math.min` и `Math.max` находятся границы массива.
- Среднее округляется `Math.round`.
- Для пустого массива возвращается `null`.

### 6. JSON (`jsonRoundTrip`)

- Объект сериализуется в строку через `JSON.stringify`.
- Строка десериализуется обратно через `JSON.parse`.
- Возвращаются и строка JSON, и восстановленный объект.

## 3 уровень

### 1. Мутация объекта (`mutateName`)

- Прямое присваивание `user.name = newName` меняет исходный объект.
- Это демонстрирует передачу объекта по ссылке.

### 2. Без мутации (`updateNameImmutable`)

- Возвращается новый объект через оператор расширения `...`.
- Исходный объект остаётся неизменным.

### 3. Параметр по умолчанию (`greetUser`)

- Если аргумент не передан, берётся `"Гость"`.
- Функция возвращает готовое приветствие.

### 4. Колбэк (`applyOperation`)

- Проверяется, что `callback` действительно функция.
- После проверки вызывается `callback(a, b)` и возвращается результат.

### 5. Цепочка областей видимости (`buildScopeMessage`)

- Вложенные функции формируют closure.
- Внутренняя функция использует переменные из `global`, `outer` и своей локальной области.

### 6. Строгий режим и калькулятор (`strictCalculator`)

- Включён `"use strict"` внутри функции.
- Через `switch` поддерживаются операции: `+`, `-`, `*`, `/`, `mod`, `pow`, `div`.
- Для операций деления добавлена проверка деления на ноль.
