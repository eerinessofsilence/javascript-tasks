# Пояснения к решениям (tier_2)

Ниже кратко объяснена логика каждого решения из `solutions/tier_2/solutions.js`.

## 1 уровень

### 1. Закипание воды (`boilingTimeSeconds`)

- Считается разница температур: `targetTemp - startTemp`.
- Каждый градус набирается за `2` секунды, поэтому итог: `delta * 2`.
- Если вода уже не холоднее целевой температуры, возвращается `0`.

### 2. Пять строк из нулей (`numberedZeroLines`)

- В цикле от `1` до нужного количества строк формируются элементы вида `"номер. 000"`.
- Результат возвращается как массив строк.

### 3. Прямоугольный треугольник (`rightTriangle`)

- На каждой итерации добавляется строка из `*` длиной `i`.
- Затем строки объединяются через `\n` для красивого вывода в консоль.

## 2 уровень

### 1. Коробка и дверь (`canBoxPassDoor`)

- Проверяются все три возможные грани коробки: `[A,B]`, `[A,C]`, `[B,C]`.
- Для каждой грани и двери стороны сортируются по возрастанию.
- Если хотя бы одна грань помещается (`x <= doorMin` и `y <= doorMax`), возвращается `"Да"`, иначе `"Нет"`.

### 2. Равнобедренный треугольник (`isoscelesTriangle`)

- Для каждой строки считаются пробелы слева: `height - i`.
- Число звёзд в строке: `2 * i - 1`.
- Это даёт симметричный треугольник по центру.

### 3. Квадраты меньше `N` (`squaresLessThanN`)

- В цикле перебираются натуральные числа `i`.
- Пока `i * i < N`, квадрат добавляется в массив.
- Возвращается список всех подходящих квадратов.

## 3 уровень

### 1. Мороженое по 3 и 5 (`canBuyIceCreamBalls`)

- Перебирается количество порций по `5`.
- Для остатка проверяется делимость на `3`.
- Если найдено подходящее разложение, ответ `"Да"`, иначе `"Нет"`.

### 2. Вклад (`yearsToExceedDeposit`)

- Сумма вклада ежегодно увеличивается по формуле: `amount += amount * k / 100`.
- Цикл продолжается, пока сумма не станет больше `s`.
- Счётчик итераций даёт количество лет.

### 3. Сумма цифр (`digitSum`)

- Берётся модуль числа, чтобы корректно обработать отрицательные значения.
- В цикле последняя цифра добавляется в сумму через `% 10`.
- Число уменьшается через `Math.floor(number / 10)`.

## Задачи на работу со списком через цикл

### 1. Сумма всех элементов (`sumArrayElements`)

- Обычный цикл `for` проходит по массиву и накапливает сумму.

### 2. Сумма чётных элементов (`sumEvenElements`)

- В цикле добавляются только элементы, для которых `item % 2 === 0`.

### 3. Минимум и максимум (`minMaxInArray`)

- Начальные `min` и `max` берутся из первого элемента.
- В одном проходе по массиву значения обновляются при необходимости.
- Для пустого массива возвращается `null`.

### 4. Индексы минимума и максимума (`minMaxIndices`)

- Хранятся индексы `minIndex` и `maxIndex`.
- В цикле сравниваются текущие элементы с элементами по этим индексам.
- Для пустого массива возвращается `null`.

## Задачи на методы списка

### 1. Список покупок (`buildShoppingList`)

- Список начинается пустым и заполняется через `push`.
- Ненужные продукты удаляются через `filter`.
- Возвращается объект с итоговым списком и количеством пунктов.

### 2. Есть ли число в списке (`hasNumber`)

- Используется `includes`, результатом является `true` или `false`.

### 3. Сколько раз встречается число (`countOccurrences`)

- Массив фильтруется по значению.
- Количество вхождений равно длине отфильтрованного массива.
